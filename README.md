# プログラミング体験会 2026 - システムリポジトリ

## 1. 動作環境（前提条件）

開発環境を構築する前に、ローカルPCに以下のソフトウェアがインストールされていることを確認してください。

- **Node.js** (v24以上推奨)
- **npm** (Node.jsに同梱)

## 2. 環境構築（初回セットアップ）

リポジトリをクローン後、プロジェクトのルートディレクトリで以下のコマンドを実行し、依存パッケージをインストールします。

```bash
npm install
```

npm workspacesを利用しているため、上記コマンドのみでフロントエンド・バックエンド両方のパッケージが一括でインストールされます。frontend/ や backend/ ディレクトリに移動して個別にインストールを実行する必要はありません。

## 3. 開発サーバーの起動
ローカルでの開発作業を開始する際は、ルートディレクトリで以下のコマンドを実行します。

```bash
npm run dev
```
上記コマンドを実行すると、concurrently によってフロントエンドとバックエンドの開発サーバーが同時に起動します。

ターミナル上で起動完了のログが表示されたら、ブラウザから以下のURLにアクセスして動作を確認してください。

- **フロントエンド（参加者用画面）:** `http://localhost:5173`
- **バックエンド（APIサーバー）:** `http://localhost:8787`

## 4. 主要なディレクトリ構成
- **frontend/:** フロントエンドアプリケーション（React + TypeScript + Vite + CSS Modules）
- **backend/:** バックエンドAPIサーバー（Cloudflare Workers + Hono）

## 5. 開発の進め方

本プロジェクトでは、`main` ブランチへの直接プッシュは禁止されています。すべての開発は以下の手順に沿って進めてください。

### ① 作業用ブランチの作成
タスクに着手する前に、必ず最新の `main` ブランチから作業用のブランチを切り出します。
ブランチ名は `feature/作業内容` や `fix/バグ内容` としてください。

```bash
# mainブランチを最新にする
git checkout main
git pull

# 新しいブランチを作って切り替える
git checkout -b feature/your-task-name
```

### ② 実装とこまめなコミット
作業の区切りごとにコミットを行います。コミットメッセージは、何をしたかが他人に伝わるように以下の「コンベンショナルコミット」のルール（接頭辞）を意識して簡潔に記述してください。

* **`feat:`** 新しい機能の追加（例: `feat: お絵かき画面のレイアウト作成`）
* **`fix:`** バグや表示崩れの修正（例: `fix: ボタンがクリックできない不具合を修正`）
* **`docs:`** ドキュメントやREADMEの修正（例: `docs: READMEに開発ルールを追記`）
* **`style:`** コードの意味を変えない、見た目の微調整やフォーマット（例: `style: インデントの修正`）
* **`refactor:`** バグ修正や機能追加を含まない、コードの整理・リファクタリング

```bash
git add .
git commit -m "feat: お絵かき画面のレイアウト作成"
```

### ③ リモートリポジトリへプッシュ
実装が完了、または途中でレビューを受けたい場合は、自分のブランチをGitHubにプッシュします。

```bash
git push origin feature/your-task-name
```
### ④ Pull Request (PR) の作成とマージ
1. GitHub上の画面から Pull Request を作成します。
2. PRのタイトルには作業内容を、説明欄には何を実装したかを記述してください。
3. レビューを受け、問題がなければ main ブランチへマージします。
