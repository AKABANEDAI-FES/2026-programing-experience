# バックエンド要件定義書

このドキュメントは、APIサーバー（Cloudflare Workers）が担うべき役割とインフラの要件を定義します。具体的なAPIの通信ルールについては `backend-api-spec.md` を参照してください。

## 1. 採用技術・インフラ
* **実行環境**: Cloudflare Workers
* **データ保存先**: Cloudflare R2 (オブジェクトストレージ)
* **リアルタイム通信**: WebSocket (または Cloudflare Durable Objects / SSE を検討)
* **言語**: TypeScript (Honoフレームワークを使用)

## 2. サーバーの主な役割
バックエンドは以下の3つの役割を確実にこなす必要があります。

1. **データの受け口 (API)**
   - フロントエンドから送られてくるJSONデータ（描画モード、画像データ、プログラム）を欠損なく受け取る。
2. **データの永続化 (Storage)**
   - 受け取ったデータを Cloudflare R2 に保存する。
   - 保存する際、後から取り出しやすいように「タイムスタンプ」や「一意のID（UUID）」をファイル名に付与して整理する。
3. **大画面へのリアルタイム通知 (WebSocket)**
   - R2への保存が完了した直後に、大画面モニターへ「新しいデータが保存された」という通知をプッシュ送信する。

## 3. セキュリティ・制約事項
* **CORS設定**: フロントエンドのURL（開発環境は `http://localhost:5173` と `http://127.0.0.1:5173`、本番環境は後日決定）からのアクセスのみを許可する。
* **エラーハンドリング**: R2への保存に失敗した場合や、不正なデータが送られてきた場合は、適切にエラー(500や400)をフロントエンドに返し、システムをクラッシュさせないこと。
