# 技術選定・採用ライブラリ一覧

本プロジェクトでは、開発スピードとパフォーマンス、そして保守性を両立させるため、以下の技術スタックとライブラリを採用（または推奨）します。

## 1. 全体基盤
* **言語**: TypeScript
  * 理由: 型があることで、エディタ（VSCode）の自動補完が効き、実行前のエラー発見が容易になるため。タイポ防止にも大きく貢献します。
* **パッケージマネージャー**: npm + npm workspaces (モノレポ構成)

## 2. フロントエンド（参加者画面・大画面）
* **フレームワーク**: React (Vite環境)
* **状態管理**: React標準フック (`useState`, `useContext`)
  * 理由: Redux等の外部ライブラリは学習コストが高すぎるため、今回はReact標準の機能のみでシンプルに状態を引き回します。

### 【推奨】特定の機能を実現するためのライブラリ
* **ドラッグ＆ドロップ (画面③用)**: `@dnd-kit/core`
  * 理由: React向けのモダンなD&Dライブラリ。動作が軽量で、今回の「ブロックを繋げる」ようなUIを作るのに非常に適しています。
* **お絵かきの線画処理 (画面②用)**: `perfect-freehand` （または生のSVG操作）
  * 理由: CanvasAPIを避ける方針のため、マウスの軌跡から滑らかなSVGのPath（線）を生成してくれるこのライブラリが有効です。
* **CSSスタイリング**: `CSS Modules` または `Tailwind CSS`

## 3. バックエンド（API・通信）
* **インフラ**: Cloudflare Workers
* **フレームワーク**: Hono
  * 理由: 軽量かつ超高速で、Cloudflare Workersとの相性が抜群です。書き方がシンプルなので、バックエンド初学者でも直感的にAPIを作成できます。
* **データベース/ストレージ**: Cloudflare R2
  * 理由: 描かれたキャラクター画像（SVG/Base64など）を安価かつ高速に保存するため。
* **リアルタイム通信**: Cloudflare Durable Objects + WebSocket
  * 理由: サーバーレス環境であるWorkers上で、「大画面との常時接続」を維持するための標準的で強力な構成です。
